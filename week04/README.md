# 📘 Today I Learned

### 1. 오늘 배운 내용
- `fetch`를 사용해서 외부 API에서 데이터를 불러오는 방법을 연습했다.
- 서버에서 바로 결과가 오는 것이 아니라, 요청을 보내고 응답을 기다린 뒤 화면을 바꾸는 흐름을 구현했다.
- API에서 받은 데이터를 그대로 쓰는 것이 아니라, 내가 만든 카드 구조에 맞게 객체 형태로 변환해서 사용했다.
- 명단 데이터가 바뀔 때마다 카드, 상세 정보, 총 인원 수가 함께 바뀌도록 렌더링 함수를 구성했다.
- 로딩 중, 완료, 실패 상태에 따라 버튼과 문구가 달라지도록 만들었다.
- 파트 필터, 이름 검색, 정렬 기능을 추가해서 같은 데이터라도 조건에 따라 다르게 보여줄 수 있게 했다.

### 2. 핵심 정리 (내 언어로)
- 이번 과제의 핵심은 “데이터가 바뀌면 화면도 같이 바뀌게 만드는 것”이었다.
- 이전에는 내가 직접 입력한 값으로만 카드를 추가했다면, 이번에는 외부 API에서 받아온 데이터를 사용했다는 점이 달랐다.
- `fetch`는 서버에 데이터를 요청하는 역할을 하고, `await`는 그 요청이 끝날 때까지 기다리는 역할을 한다.
- API에서 받은 데이터는 내가 원하는 형태와 다르기 때문에, `map`을 사용해서 화면에서 쓰기 좋은 객체 형태로 바꿔야 했다.
- 화면을 직접 하나씩 수정하기보다, 배열 데이터를 기준으로 전체 화면을 다시 그리는 방식이 더 이해하기 쉬웠다.
- 로딩이나 실패 상태를 화면에 보여줘야 사용자가 지금 무슨 일이 일어나고 있는지 알 수 있다는 것도 느꼈다.
- 필터, 검색, 정렬은 원본 데이터를 바꾸는 것이 아니라, 현재 조건에 맞는 데이터만 골라서 보여주는 방식으로 구현했다.

### 4. 느낀 점
이번 과제를 하면서 비동기 처리가 왜 필요한지 조금 더 직접적으로 이해할 수 있었다. 버튼을 누른다고 바로 결과가 나오는 것이 아니라, 서버에 요청을 보내고 응답을 기다린 뒤 그 데이터를 화면에 반영해야 했다. 처음에는 단순히 카드를 추가하는 기능처럼 보였지만, 로딩 상태나 실패 상황까지 생각하니까 코드의 흐름을 더 신경 써야 했다. 특히 API에서 받아온 데이터를 바로 화면에 넣는 것이 아니라, 내가 만든 카드 구조에 맞게 다시 정리하는 과정이 중요하다고 느꼈다. 데이터 배열을 기준으로 화면을 다시 그리면 추가, 삭제, 검색, 정렬 같은 기능을 한 번에 관리하기가 훨씬 편했다. 아직 코드가 완전히 익숙한 것은 아니지만, 이번 과제를 통해 데이터와 UI가 어떻게 연결되는지 조금은 감을 잡을 수 있었다.
