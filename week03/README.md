# 📘 Today I Learned

### 1. 오늘 배운 내용
- 버튼을 클릭했을 때 폼이 열리고 닫히는 기능을 만들어봤다.
- 폼에 입력한 값을 JavaScript로 가져와서 새로운 카드로 추가했다.
- 아기 사자 정보를 배열 안에 객체 형태로 저장해서 관리했다.
- 배열에 데이터가 추가되거나 삭제될 때 카드 목록과 총 인원 수도 같이 바뀌도록 했다.
- 빈칸이 있거나 이메일, 웹사이트 형식이 맞지 않으면 안내 문구가 나오도록 했다.

### 2. 핵심 정리 (내 언어로)
- HTML에 적혀 있는 내용만 화면에 보이는 줄 알았는데, JavaScript로도 화면을 새로 만들 수 있다는 걸 알게 됐다.
- 이번 과제에서는 아기 사자 한 명의 정보를 하나의 객체로 만들고, 여러 명의 정보는 배열에 담아서 관리했다.
- 추가 버튼을 누르면 입력값을 읽고, 그 값을 새 객체로 만든 다음 배열에 넣었다.
- 삭제 버튼을 누르면 배열에서 마지막 데이터를 지우고 화면도 다시 그리게 했다.
- 결국 화면을 직접 하나하나 고치는 게 아니라, 데이터가 바뀌면 화면도 다시 그려지게 만드는 방식이라고 이해했다.

### 3. 결과 이미지(스크린샷)
- 처음 화면에는 아기 사자 6명의 카드가 보이도록 만들었다.
- 상단에는 추가 버튼, 삭제 버튼, 총 인원 수가 보인다.
- 추가 버튼을 누르면 입력 폼이 나타나고, 다시 누르면 폼이 닫힌다.
- 정보를 입력하고 추가하기를 누르면 요약 카드와 상세 카드가 아래에 새로 생긴다.
- 마지막 삭제 버튼을 누르면 가장 마지막 카드가 사라지고 총 인원 수도 같이 줄어든다.
- 화면이 좁아지면 카드와 입력창이 한 줄씩 아래로 쌓이게 했다.

### 4. 느낀 점
이번 과제는 처음에 조금 헷갈렸다. HTML이랑 CSS로 화면을 만드는 건 어느 정도 익숙했는데, JavaScript로 데이터를 추가하고 그 결과가 화면에 바로 반영되게 만드는 부분이 어려웠다. 그래도 코드를 나눠서 보니까 흐름이 조금 보였다. 먼저 HTML에 있던 기존 카드 정보를 배열로 가져오고, 폼에 입력한 값을 새 데이터로 만든 다음, 그 데이터를 기준으로 화면을 다시 그리는 방식이었다.특히 `배열에 데이터가 들어가면 카드가 생기고, 배열에서 데이터가 빠지면 카드도 사라진다`는 흐름이 중요하다고 느꼈다. 아직 JavaScript 문법이 완전히 익숙한 건 아니지만, 이번 과제를 하면서 버튼 클릭이나 폼 제출이 실제 화면 변화로 이어지는 과정을 조금은 이해하게 된 것 같다.
