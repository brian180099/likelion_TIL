# 📘 Today I Learned

### 1. 오늘 배운 내용
- Google Fonts를 활용해 웹페이지에 원하는 폰트를 적용하는 방법을 배웠다.
- Font Awesome을 활용해 아이콘을 웹페이지에 넣는 방법을 배웠다.
- HTML 문서의 `head` 영역에 메타 정보, CSS 파일, 외부 리소스를 연결하는 방법을 학습했다.
- `body` 안에서 `nav`, `header`, `section`, `footer`를 사용해 웹페이지를 구조적으로 나누는 방법을 배웠다.
- 웹페이지를 만들 때 단순히 태그를 나열하는 것이 아니라, 각 영역의 역할에 맞는 시맨틱 태그를 사용하는 것이 중요하다는 점을 이해했다.

### 2. 핵심 정리 (내 언어로)
- Google Fonts는 웹페이지 글꼴을 더 보기 좋게 바꿔주는 외부 폰트 서비스이고, `head`에 링크를 연결한 뒤 CSS에서 `font-family`로 적용할 수 있다.
- Font Awesome은 다양한 아이콘을 쉽게 넣을 수 있게 해주는 라이브러리로, 별도의 이미지 없이도 아이콘을 사용할 수 있어서 편리하다.
- `head`는 화면에 직접 보이지는 않지만, 문서의 설정 정보와 외부 파일 연결을 담당하는 중요한 영역이다.
- `nav`는 메뉴 영역, `header`는 페이지의 핵심 소개 영역, `section`은 본문 내용을 주제별로 나누는 영역, `footer`는 하단 마무리 영역이다.
- 이번 학습을 통해 HTML은 웹페이지의 구조를 만들고, CSS는 그 구조를 꾸미는 역할을 한다는 점을 다시 정리할 수 있었다.
- 결국 좋은 웹페이지는 단순히 예쁘게 만드는 것이 아니라, 구조가 명확하게 보이도록 작성하는 것이 중요하다고 느꼈다.

### 3. 느낀 점
이번 실습을 하면서 웹페이지를 만들 때 각 태그를 아무렇게나 사용하는 것이 아니라, 역할에 맞게 구조를 나누는 것이 정말 중요하다는 점을 느꼈다.  
특히 `nav`, `header`, `section`, `footer`처럼 의미를 가진 태그를 사용하면 코드가 더 깔끔해지고, 나중에 다시 봤을 때도 구조를 이해하기 쉬웠다.  
또한 Google Fonts와 Font Awesome 같은 외부 리소스를 활용하면 복잡한 작업 없이도 웹페이지의 완성도를 높일 수 있다는 점이 인상적이었다.  
앞으로는 단순히 화면을 만드는 것에 그치지 않고, 구조적으로도 잘 정리된 HTML 문서를 작성하는 연습을 더 하고 싶다.
